# Contributing

## Pull requests checklist

- [x] Branch created from `main` and named using conventions
  - (e.g. `feat/short-description`, `fix/short-description`).
- [x] Commits are atomic and follow conventional commits
  - (e.g. `feat: add X`, `fix: correct Y`, `chore: update deps`).
- [x] Local linting, formatting and TypeScript type checks passed:
  - `npm run lint`, `npm run format` and `npm run typecheck`.
- [x] Unit and integration tests added or updated where applicable;
  - existing tests pass: `npm run test`.
- [x] Build succeeds: - `npm run build`.
- [x] Updated documentation or README if public behavior changed.
- [x] Screenshots attached for UI changes.
- [x] No secrets or credentials in the diff. Use GitHub Secrets or host secret stores.

> Add short notes explaining the motivation and any migration steps needed for reviewer.

Thanks for contributing! See package-specific READMEs for details.
